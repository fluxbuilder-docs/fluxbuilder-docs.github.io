FluxBuilder v2.4.0 delivers powerful enhancements that give you **more design freedom**, **better localization control**, and **stronger shopping experiences** across mobile apps and PWAs.



---


## (1) Important Updates 💥

1. Remove **Firebase** **Dynamic Link** completely
2. Remove the old Shopify Checkout API completely (now use the Cart API as default)
3. In the [Web version](https://web.fluxbuilder.com/), Image quality may be reduced to help FluxBuilder load faster. Rest assured, your real app will display full-resolution visuals. 
Or you can try the [Desktop version](https://fluxbuilder.com/download) for the best visual experience.

## (2) Troubleshoot with AI 🤖


FluxBuilder now supports **AI-assisted troubleshooting** to help you identify and understand issues faster. The AI analyzes system checks and error details to provide **clear insights and suggestions**, reducing debugging time and making problem-solving easier—even for non-technical users.


👉 [Please review this step-by-step guide for creating a new app before troubleshooting.](https://docs.fluxbuilder.com/create-your-app)


![](/images/notion/e73521fc7246.png)


## (3) API Call Logs with Status Codes (Troubleshoot Wizard) 💫


FluxBuilder now records **API call logs** inside the Troubleshoot Wizard, including response status codes (for example: 200 success, 404 error) and error details when available. These logs are included in the **Copy Logs** output for easier debugging and support investigation, while not being shown directly on the UI, keeping the interface clean and secure.


![](/images/notion/cb74a77350e7.png)


## (4) Screen Duplication Button 🖱️


Instantly duplicate any existing screen with one click. This helps you build apps faster, reuse layouts efficiently, and reduce repetitive setup when creating similar pages or flows.

## (10) WebView: Smart Domain Detection **🔬**


FluxBuilder now intelligently handles link navigation based on domain rules. 

- **External Domains** will open in the device’s default browser, while **Internal Domains** can continue to open inside the app WebView.
- This prevents broken flows, improves checkout and login reliability, and gives you full control over user navigation.

![](/images/notion/84839967f191.png)
